Let’s learn some basic ideas about the MVC pattern

10 April

The MVC Pattern

  • Model-View-Controller(MVC) has been an important architectural pattern in computer science for many years
  • Separates concerns within application
  • Utilized by dozens of frameworks

Models

  • A set of classes that describe the data you will be working with

View

  • Defines how the application’ UI will be displayed
  • This is a template to dynamically generate HTML

Controller

  • Sits between Model and View that handles communication from user, application flow and code logic
  • Responds to user input, talks to the Model, and decides which view to render if any
  • Think of this class as where all of your code statements go (business logic)

Razor View Engine

  • Razor is code syntax used by the view
  • Used for HTML template generation
  • Does not use the heavy XML syntax used by the Web Forms view engine
  • Minimal Keystrokes and requires the use of pure html and JavaScript

Unobtrusive JavaScript

  • Place JavaScript code within JavaScript and jQuery libraries of your own design and reduces clutter of JavaScript code with scripts

jQuery Validation

  • Validation done using jQuery Validation plugin
  • Reduce the writing of validation code
  • Validations are performed using Unobtrusive JavaScript

JSON Binding

  • Automatically includes JavaScript Object Notation(JSON) binding support enabling action methods to accept and model-bind data in JSON format
  • Templates and data binding that need to post data back to the server

Dependency Resolution

  • Simplifies the use of dependency injection in your applications
  • Easier to decouple application components, making them more configurable and easier to test

Display Modes

  • Real-time selection of display modes based on hardware device used
  • Index.cshtml – renders for desktop screens
  • Index.Mobile.cshtml – renders for mobile device screens

Bundling & Minification

  • Reduces requests to your sites by combining several individual script references into a single request
  • Minifies the requests through a number of techniques, such as shortening variable names and removing white spaces and comments
