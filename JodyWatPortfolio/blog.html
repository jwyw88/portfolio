HTML5 – Canvas

07 April

According to W3C, the Canvas is:

  • The HTML5 Canvas element is used to draw graphics, on the fly, via scripting (usually JavaScript).
  • The Canvas element is only a container for graphics. You must use a script to actually draw the graphics.
  • Canvas has several methods for drawing paths, boxes, circles, text, and adding images.

For more information, please refer to:

W3C

Browser Support

Internet Explorer 9+, Firefox, Opera, Chrome, and Safari


Canvas Coordinates

  • Two-dimensional grid
  • Starts at upper left
  • Move right -> increase X value
  • Move left -> increase Y value


Canvas markup

A Canvas element has no content and no border of its own.


Canvas element

<canvas> width=" " height=" "> </canvas>


  • Always specify width and height of drawing area
  • Include ID attribute for JavaScript
  • Include fallback content for non-supportive browsers


JavaScript

  • Used to display drawn objects/text
  • Find the Canvas element through:

<script>
Var getCanvas = document.getElementById(“myCanvas”); -> Finds the Canvas element with id=”myCanvas”
Var ctx = getCanvas.getContext(“2d”); -> getContext() method renders context and its drawing functions
</script>


Drawing Rectangles

  • fillstyle()- CSS color, a pattern, or a gradient. The default is solid black
  • fillRect(x, y, width, height) draws a rectangle filled with the current fill style.
  • strokeStyle like fillStyle — it can be a CSS color, a pattern, or a gradient.
  • strokeRect(x, y, width, height) draws a rectangle with the current stroke style. ->it doesn’t fill in the middle; it just draws the edges.
  • clearRect(x, y, width, height) clears the pixels in the specified rectangle.
